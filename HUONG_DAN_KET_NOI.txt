================================================================================
HƯỚNG DẪN KẾT NỐI — TrolyWP Agent Client + n8n + WEBO MCP
================================================================================
(Mọi URL, domain, key, token trong tài liệu chỉ là data sample — thay bằng giá trị thật của bạn.)


1. TỔNG QUAN QUY TRÌNH
--------------------------------------------------------------------------------

  [User] → [WordPress] → [n8n] → [WordPress MCP] (khi AI gọi tool)

  • Chat: Trình duyệt gọi thẳng n8n (webhookUrl = URL Chat Trigger). WordPress chỉ inject meta (firstEntryJson: site, user, chat_token, authorKey) vào trang; widget @n8n/chat gửi kèm meta đó. Không proxy chat, không HMAC từ trình duyệt (secret chỉ trên server).
  • Tool MCP: Khi AI trong n8n cần gọi WordPress, có 2 cách:
      A) Proxy (khuyến nghị): n8n gửi qua mcp-proxy với chat_token; WordPress ký HMAC theo từng user — mỗi author dùng đúng key, không cần lưu Secret trong n8n.
      B) HMAC trực tiếp: n8n dùng credential với Key ID + Secret cố định (một user/key cho mọi request).


2. THÀNH PHẦN
--------------------------------------------------------------------------------

  • webo-wordpress-mcp   — MCP router trên WordPress (nhận JSON-RPC: initialize, tools/list, tools/call).
  • webo-hmac-auth       — Cấp key HMAC cho user, ký request (chuẩn: METHOD + path + ts + sha256(body)).
  • trolywp-agent-client — Chat widget (@n8n/chat), inject meta (firstEntryJson) từ server; MCP proxy (chat_token → HMAC). Không proxy chat.
  • n8n-nodes-webo-mcp   — Node n8n: WEBO MCP (tools/list, initialize, tools/call). Credential: HMAC hoặc Proxy (theo author).


3. CÀI ĐẶT TRÊN WORDPRESS
--------------------------------------------------------------------------------

  3.1. Cài và kích hoạt (theo thứ tự)
        — webo-wordpress-mcp (MCP router).
        — webo-hmac-auth (Network Activate nếu multisite).
        — trolywp-agent-client (phụ thuộc webo-hmac-auth).

  3.2. Tạo HMAC key cho user
        — Vào Network Admin → Users → Edit User (hoặc Users → Edit User nếu single site).
        — Phần "WEBO API Keys (HMAC)": tạo key, lưu Key ID và Secret (Secret chỉ hiện một lần).
        — Multisite: user network đăng nhập child site vẫn dùng được key (authorKey có trong firstEntryJson).

  3.3. Cấu hình TrolyWP Agent Client
        — WP Admin → TrolyWP Client → Cài đặt.
        — Chế độ n8n: "Dùng n8n riêng (tùy chỉnh)".
        — Chat URL: URL từ node Chat Trigger trong n8n (data sample: https://n8n.example.com/webhook/abc123/chat).
        — Lưu.


4. CÀI ĐẶT TRÊN N8N
--------------------------------------------------------------------------------

  4.1. Chat Trigger
        — Workflow có node "Chat Trigger" (@n8n/chat).
        — Lấy Production URL (hoặc Test URL) làm Chat URL nhập vào WordPress (mục 3.3).
        — Allowed Origins: * hoặc thêm domain site WordPress (data sample: https://your-wordpress-site.com).

  4.2. Credential WEBO MCP API (chọn 1 trong 2)

        A) Proxy (theo author) — khuyến nghị khi chat từ nhiều user/site
           — Authentication: chọn "Proxy (theo author: chat_token)".
           — Base URL: có thể để trống (node lấy site_url từ input).
           — Không điền Key ID / HMAC Secret.
           — Input vào node WEBO MCP phải có metadata từ chat: metadata.metadata.site_url (hoặc metadata.site_url) và metadata.chat_token. Chat Trigger trả về đúng format này.

        B) HMAC trực tiếp — 1 key/secret cho mọi request
           — Authentication: chọn "HMAC (webo-hmac-auth)".
           — Base URL (data sample): https://your-wordpress-site.com/wp-json/mcp/v1/router
           — Key ID + HMAC Secret: điền key đã tạo tại Edit User (WordPress).

  4.3. Workflow AI Agent
        — Kết nối node WEBO MCP vào AI Agent (tool).
        — Node WEBO MCP: Name (tools/list | initialize | tên tool), Arguments (JSON).
        — Nếu dùng Proxy: đảm bảo input của WEBO MCP đến từ item có metadata (vd từ Chat Trigger); node tự lấy site_url + chat_token để gọi mcp-proxy.
        — Prompt hệ thống: gọi tools/list trước để lấy danh sách tool, sau đó gọi tool với name + arguments (domain, author, ...).


5. CÁCH SỬ DỤNG THEO TỪNG CÁCH AUTH (DATA SAMPLE)
--------------------------------------------------------------------------------

  5.1. Dùng Proxy (HMAC thay đổi theo author)

        • Trên WordPress: user đăng nhập, có HMAC key (authorKey sẽ có trong firstEntryJson).
        • Chat Trigger n8n nhận message kèm metadata — data sample:
            metadata.metadata.site_url = "https://your-wordpress-site.com/"
            metadata.chat_token         = "a1b2c3d4e5f6... (hex, sliding TTL 15 phút — mỗi lần gọi MCP thành công thì gia hạn, không cần logout)"
            metadata.authorKey          = "wk_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
        • Credential WEBO MCP API: Auth = "Proxy (theo author: chat_token)", Base URL để trống.
        • Trong workflow: output của Chat Trigger (hoặc node trước WEBO MCP) phải chứa metadata trên; nối vào node WEBO MCP. Node sẽ POST tới {{site_url}}/wp-json/trolywp-client/v1/mcp-proxy với body { chat_token, body }.
        • WordPress mcp-proxy kiểm tra chat_token → user → ký HMAC bằng key của user đó → gọi MCP router. Mỗi author tự động dùng đúng key.

  5.2. Dùng HMAC trực tiếp (1 key cố định)

        • Credential: Auth = HMAC, Base URL = MCP router (vd https://your-wordpress-site.com/wp-json/mcp/v1/router), Key ID + Secret = key một user (data sample Key ID: wk_xxxxxxxx...).
        • Mọi request từ n8n đều dùng key đó. Phù hợp khi chỉ một site/một user hoặc không dùng chat_token.


6. LUỒNG CHAT (REQUEST TỪ TRÌNH DUYỆT)
--------------------------------------------------------------------------------

  Bước 1: User mở trang có chat (đã đăng nhập WordPress).
  Bước 2: WordPress inject script: webhookUrl = URL n8n (Chat URL đã cấu hình), metadata = firstEntryJson (site, user, chat_token, authorKey...).
  Bước 3: Widget @n8n/chat gửi request thẳng tới n8n (webhookUrl), kèm metadata trong body theo chuẩn chat n8n.
  Bước 4: n8n nhận request (có firstEntryJson trong metadata), xử lý chat, trả kết quả về trình duyệt.

  → Không qua proxy. Meta do WordPress tạo trên server và đưa vào config widget; trình duyệt không có HMAC secret.


7. LUỒNG MCP (N8N GỌI WORDPRESS)
--------------------------------------------------------------------------------

  Khi AI trong n8n cần gọi tool (vd tạo bài viết):

  Cách A — Proxy (node WEBO MCP với Auth = Proxy)
  — Node lấy từ input: metadata.metadata.site_url (hoặc metadata.site_url), metadata.chat_token.
  — Node gửi POST tới {{site_url}}/wp-json/trolywp-client/v1/mcp-proxy, body (data sample): { "chat_token": "a1b2c3d4...", "body": { "jsonrpc": "2.0", "method": "tools/call", ... } }.
  — WordPress mcp-proxy: chat_token → user_id → key của user → ký HMAC → POST /wp-json/mcp/v1/router.
  — Kết quả trả về n8n. n8n không cần lưu Secret.

  Cách B — HMAC trực tiếp
  — Node gửi POST thẳng tới Base URL (MCP router), header HMAC lấy từ credential (Key ID + Secret).

  Bước chung sau khi có response: webo-wordpress-mcp + webo-hmac-auth xác thực và thực thi MCP, trả kết quả.


8. CHUẨN HMAC (webo-hmac-auth)
--------------------------------------------------------------------------------

  baseString = METHOD + "\n" + path + "\n" + ts + "\n" + sha256(body_raw)
  Header: X-WEBO-KEY, X-WEBO-TS, X-WEBO-SIGN (base64 HMAC-SHA256)

  Path (data sample): /wp-json/mcp/v1/router. Body: raw string (POST).


9. TÓM TẮT URL (DATA SAMPLE)
--------------------------------------------------------------------------------

  Trên WordPress:
  — MCP proxy (n8n gọi):           https://your-wordpress-site.com/wp-json/trolywp-client/v1/mcp-proxy
  — MCP router (WordPress nội bộ): https://your-wordpress-site.com/wp-json/mcp/v1/router

  Trên n8n:
  — Chat Trigger URL (data sample): https://n8n.example.com/webhook/your-webhook-id/chat → nhập vào WordPress (Cài đặt) làm Chat URL. Chat gọi thẳng URL này.


10. XỬ LỖI THƯỜNG GẶP
--------------------------------------------------------------------------------

  — Chat không load: Kiểm tra Chat URL đúng Production/Test URL của Chat Trigger; Allowed Origins có domain site (vd https://your-wordpress-site.com).
  — authorKey rỗng: User chưa có key HMAC hoặc key chưa gán user meta; tạo key tại Edit User (webo-hmac-auth). Multisite: user network đăng nhập child site vẫn dùng được nếu đã tạo key cho user đó.
  — MCP proxy 401: chat_token hết hạn (sau 15 phút không có request MCP) hoặc sai; firstEntryJson phải có chat_token. Token dùng sliding expiry: mỗi lần gọi proxy thành công thì gia hạn thêm 15 phút — user không cần logout. Muốn đổi TTL: filter trolywp_chat_token_ttl (giây).
  — user_has_no_hmac_key: User chưa có key HMAC trong webo-hmac-auth; tạo key tại Edit User.
  — Proxy auth "requires site_url and chat_token": Input item vào node WEBO MCP thiếu metadata.site_url hoặc metadata.chat_token; kiểm tra output Chat Trigger có truyền metadata xuống node WEBO MCP.
  — Form credential chỉ có "HMAC", không thấy "Proxy (theo author)": Node trên server chưa cập nhật. Chạy lại deploy.ps1 (copy vào ~/.n8n/custom), restart n8n, tạo lại credential mới (hoặc xóa credential cũ và thêm lại).
  — Dùng expression domain / hmac_sign từ InputChat: WordPress không gửi domain hay hmac_sign; chỉ gửi metadata.site_url, metadata.chat_token, authorKey. Để HMAC thay đổi theo author phải chọn Auth = Proxy, không dùng HMAC + expression secret.
